# Лендінг курсу подології

Готовий статичний лендінг (один файл `index.html`, без залежностей).

## Як завантажити на сервер
Просто скопіюйте `index.html` у корінь хостингу. Працює на будь-якому:
- звичайний хостинг / FTP — покладіть файл у `public_html`;
- GitHub Pages, Netlify, Vercel — перетягніть файл/папку;
- відкривається і локально подвійним кліком.

## Що замінити перед запуском (пошук по тексту `ВАШ_АКАУНТ` та `TODO`)
1. **Instagram** — `https://instagram.com/ВАШ_АКАУНТ` → ваш профіль (кілька місць).
2. **Telegram** — `https://t.me/ВАШ_АКАУНТ` → ваш акаунт або `https://t.me/+380XXXXXXXXX`.
3. **Блок «Викладач»** — замініть `[Ім'я...]`, опис і цифри в `t-stats`.
4. **Фото** — зараз стоять акуратні заглушки (блоки з підписами `Фото...`).
   Щоб поставити реальні фото — створіть папку `assets/` і замініть `<div class="ph">...</div>`
   на `<img src="assets/hero.jpg" alt="...">` (так само для викладача).
5. **OG-зображення** (для красивого прев'ю у соцмережах) — розкоментуйте `og:image` у `<head>`
   і покладіть `assets/og-cover.jpg` (рекоменд. 1200×630).

## Meta Ads / Pixel
- У `<head>` є закоментований блок **Meta Pixel** — вставте свій `PIXEL_ID` і розкоментуйте.
- Унизу в `<script>` є закоментований трекінг події **Lead** на кліки CTA — розкоментуйте після підключення Pixel.

## Структура екранів
Офер → Користь → Програма (День 1 онлайн / День 2 офлайн) → Викладач → Сертифікат →
Ціна та дата → FAQ → фінальний заклик. Липкий хедер + плаваюча панель CTA на мобільному.

## Воркфлоу: швидко зробити новий лендінг із цього шаблону

Цей репозиторій позначено як **template**, тож із нього можна «штампувати» нові лендінги.
Потрібен встановлений [GitHub CLI](https://cli.github.com/) (`gh auth login`).

```powershell
# 1. Створити новий публічний репозиторій з копією шаблону (замініть НАЗВУ)
gh repo create landing-НАЗВА --template YanisDigital/case-site-course-podolog-ua --public --clone

# 2. Зайти в папку та відредагувати index.html (текст, посилання, Pixel)
cd landing-НАЗВА

# 3. Запушити зміни
git add -A
git commit -m "Edit content"
git push

# 4. Увімкнути GitHub Pages (гілка main, корінь)
gh api -X POST repos/YanisDigital/landing-НАЗВА/pages -f "source[branch]=main" -f "source[path]=/"

# (опц.) перевірити статус збірки та адресу
gh api repos/YanisDigital/landing-НАЗВА/pages --jq .html_url
```

Через ~1 хв сайт доступний за адресою `https://yanisdigital.github.io/landing-НАЗВА/`.

### Що міняти в кожній копії (пошук-заміна в `index.html`)
- `ВАШ_АКАУНТ` → Instagram / Telegram
- `PIXEL_ID` → ваш Meta Pixel
- текст офера, програми, ціни, дати, блок «Викладач»

### Коли краще не GitHub Pages
- Потрібен **приватний** код лендінгів → GitHub Pro (~$4/міс) або **Netlify** (безкоштовно, приватні репо, drag-and-drop, гарніші домени).
- Для платної реклами Meta солідніше виглядає **власний домен** — його можна безкоштовно прив'язати до Pages.

### Ліміти безкоштовного тарифу (на кожен сайт)
Сайтів — необмежено (1 на репозиторій). Розмір сайту до 1 ГБ, трафік 100 ГБ/міс, до 10 збірок/год.
Pages працює лише з **публічних** репозиторіїв на безкоштовному тарифі.
